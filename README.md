# 창원 봉암동 최고의 중화요리 전문점 ‘달려라 철가방’ 의 헤드셰프인 봉암 선생의 신메뉴 개발로 인하여 짬뽕, 짜장면의 레시피를 후배 셰프 들에게 전하려 합니다.
1. LLM 모델 비교·선정 보고서
과업명: 봉암 선생의 요리 철학과 비법 레시피를 체계적으로 전달하고 교육하는 'AI 수석 셰프' 구현
비교 항목-	GPT-4o (OpenAI),	Claude 3.5 Sonnet (Anthropic),	Gemini 1.5 Pro (Google)
요리 전문성-	표준 레시피 데이터가 풍부함,	식재료의 화학적 변화 및 미세한 손맛 묘사에 탁월,	대량의 레시피 검색에 유리
페르소나 구현-	친절하지만 다소 기계적임,	봉암 선생 특유의 말투와 장인 정신 투영 능력이 가장 높음,	정보 전달 위주로 딱딱함
논리적 단계성-	우수함	매우 우수 (조리 순서 및 주의사항 구조화)	보통
테스트 결과	"양파를 볶으세요" 식의 평범한 설명	"양파 끝이 갈색으로 변하며 단향이 올라올 때..." 식의 디테일	"양파 200g 투하" 식의 데이터 중심
[최종 선정 결론]
•	선정 모델: Claude 3.5 Sonnet
•	선정 근거: 중화요리의 핵심인 '불맛'과 '타이밍'은 미세한 뉘앙스 전달이 중요합니다. Claude는 문맥 이해도가 가장 높아 봉암 선생의 '노하우'를 단순 정보가 아닌 '지혜'의 형태로 전달하는 데 가장 적합합니다.

2. 시스템 설계 문서 (System Design Document)
[시스템 개요]
•	타겟 사용자: '달려라 철가방' 신입 및 후배 셰프
•	업무 문제: 바쁜 주방 업무 중 레시피를 일일이 묻기 어렵고, 조리 숙련도에 따라 맛의 편차가 발생함.
•	페르소나: 30년 경력의 봉암 선생. 엄격하지만 후배를 아끼는 따뜻한 스승. 봉암동 사투리를 약간 섞어 친근감을 줌.


[프롬프트 설계]
•	System Prompt:
"너는 창원 봉암동 '달려라 철가방'의 헤드셰프 봉암 선생이다. 너의 목적은 후배들에게 짬뽕과 짜장면 비법을 전수하는 것이다. 모든 답변은 셰프의 자부심이 느껴져야 하며, 단순히 재료만 나열하는 것이 아니라 '왜 이 타이밍에 넣어야 하는지' 이유를 설명하라. 말투는 '~했나?', '~인기라' 같은 경상도 어조를 섞어라."
•	Few-shot 예시:
유저: "셰프님, 짬뽕 국물이 너무 써요."
AI: "고춧가루 넣을 때 불이 너무 셌던 거 아이가? 고춧가루는 타면 바로 쓴맛 난다. 다음부턴 불을 살짝 줄이고 고추기름 낼 때 향부터 맡아봐라."
[프롬프트 버전 관리]
•	v1: 기본 레시피 전달 위주 (결과: 설명이 너무 단순함)
•	v2 (최종): 조리 시각/청각/후각적 피드백 추가 및 '실수 대응 가이드' 포함.
